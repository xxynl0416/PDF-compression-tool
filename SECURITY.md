# Security Policy

## Supported Versions

Security fixes are provided for the latest `main` branch and the latest tagged release.

## Reporting a Vulnerability

Please do **not** open public issues for security vulnerabilities.

Use one of the following channels:

- Open a private security advisory on GitHub (preferred)
- Or send a private report to the maintainers with:
  - Vulnerability description
  - Reproduction steps / PoC
  - Impact assessment
  - Suggested fix (if any)

## Response Process

- We will acknowledge reports within 72 hours.
- We will triage severity and provide an estimated remediation timeline.
- We will coordinate disclosure after a fix is available.

## Scope Notes

- Third-party dependency vulnerabilities may require upstream fixes.
- Misconfiguration risks (e.g., exposing debug mode in production) are in scope and should be reported.
